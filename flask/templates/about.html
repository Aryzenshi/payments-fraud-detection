The objective of this system is to predict online payment fraud given various transaction parameters. This is a classification problem where the target variable is fraud detection (categorical values). The purpose is to separate the available supply of online payments into classes differing in risk levels.

Fraud Detection Workflow

üîç
  1. Data Collection & Preprocessing: Collect transaction data, handle missing values, outliers, and encode categorical features.
  2. Feature Engineering & Model Selection: Select relevant variables and compare multiple ML models (RFC, DTC, SVC, XGBoost, Extra Trees). ETC was finalized for deployment.
  3. Model Training & Evaluation: Train on historical data and evaluate using accuracy, precision, recall, and F1-score.
  4. Model Deployment: Deploy the model to detect fraud in real-time and continuously improve using feedback loops.
